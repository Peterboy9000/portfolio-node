Nexus AI Pipeline

Serverless ML Inference System · 2022 – 2023

The previous system had p99 latency of over 4 seconds and required manual scaling intervention during traffic spikes. I redesigned the entire pipeline using AWS Lambda for inference, SQS for job queuing, and a lightweight Go service for orchestration.

Pulse Notifications

Event-Driven Omnichannel Notification Service · 2021

Pulse is a unified notification delivery service that I built to consolidate four separate notification pipelines — push, email, SMS, and in-app — into a single event-driven system. The previous setup had each channel managed independently, making it impossible to deduplicate, rate-limit, or coordinate notifications across channels.

The service is built on a Kafka backbone that decouples event producers from delivery channels, enabling reliable at-least-once delivery with idempotency guarantees. A TypeScript-based orchestration layer handles routing logic, user preferences, and per-channel rate limiting.

// Key Metrics
500k+
Messages / Day
0
Message Loss
4

// What I Built
  • Kafka-backed event pipeline with topic-per-channel architecture and consumer group auto-rebalancing
  • TypeScript orchestration service with pluggable channel adapters for WebSocket, push, email (SendGrid), and SMS (Twilio)
  • User preference engine allowing per-channel opt-outs, quiet hours, and digest scheduling
  • Idempotency layer using Redis to prevent duplicate deliveries across retries and consumer restarts
  • Admin dashboard for monitoring delivery rates, failure queues, and per-user notification history
// Outcome

Pulse now processes over 500k notifications per day with zero confirmed message loss since go-live. Unifying the four pipelines also cut operational overhead significantly — the on-call team went from managing four separate alerting systems to one, reducing incident response time by half.
